About

A few words about MetWork Framework

MetWork (as METeorological frameWORK) was originaly built by Météo-France as the core of its new weather forecasting workstation.

As this framework was successfull internaly, even for developping things far away from its original domain, it has been deciced recently to release it as an independant product under a very permissive BSD license and to maintain it through a truly open process.

  • Production ready
  • Free and Open
  • Originaly designed for Meteorology or IoT

Features

Recent softwares Linux add-on

MetWork contains a "dependencies package" ridiculously easy to install which add to an entreprise Linux distribution plenty of recent softwares, especially for scientific and meteorology domain.

Incoming files processing framework

With the "incoming files" processing framework, you can define a set of actions (you can provide as plugins) to easily build a custom but robust stream processing workflow for your incoming files.

"Geospatial ready" storage framework

With this MetWork part, by providing your schema as a plugin, you can easily build a storage system based on SQL with some geospatial extensions and/or on NoSQL services (for dealing with big binary files for example).

Webservices framework

With the "webservices framework", you are able to quickly build some REST micro-services as plugins with uptodate technologies like "asynchronous Python3", "nodejs" or with robust and standard Python2 or Python3 synchronous workers.

Webservices QOS framework

<Soon>Thanks to this part of MetWork, you can build a "production ready" webservices cluster with a kind of integrated http router to implement advanced business rules for rate limiting, throtting, priority handling...</Soon>

Monitoring framework

Last but not least, all MetWork parts can send metrics and logs to an integrated monitoring module which provide modern dashboards to diagnose what's going on your system. Of course, you can provides your own metrics and dashboards.
